Getting Started

What is Crystord?

Crystord is a Google Workspace Add-on that syncs variables between Google Sheets™ and Google Slides™. Use it to create dynamic slide templates where placeholders like {{ '{{' }}variable_name{{ '}}' }} are replaced with values from your spreadsheet.

Basic setup:

  • Install the Crystord Add-on from the Google Workspace Marketplace.
  • Open your Slides™ presentation and add your variables in any part of your slides as {{ '{{' }}variable_name{{ '}}' }}.
  • Click the bottom-left expansion arrow and press the Crystord icon to open the sidebar.
  • The sidebar will display four buttons:
    • Create Database: creates a new Sheets™ file named "database" that the Crystord Add-on is allowed to read, and opens it in a new tab. This file is automatically selected and is initially empty. Add your variables to this file as needed, by filling the table with the columns "Name" and "Value", along with your variables names and values.
    • Select Database: allows you to select a previously created Sheets™ file (database).
    • Open Database: opens the selected database in a new tab.
    • Synchronize: updates your current Slides™ file with the latest values from the selected database. This works the same way as before.
  • The first step is to create a database, as nothing works without it. When a new database is created, it is automatically selected. After making any changes to the database, always click the "Synchronize" button to ensure updates are reflected in your Slides. Additionally, use clear and consistent naming conventions for variables in the database to avoid conflicts or errors during synchronization.

Frequently Asked Questions

How do I install the add-on?

Install directly via the Google Workspace Marketplace or from within Google Slides™ side panel by searching for "Crystord".

The variables in Slides aren't updating.

  • Ensure you have created and selected a database using the "Create Database" or "Select Database" buttons in the sidebar.
  • Double-check that the variable names in your Slides match exactly with the names in the database (case-sensitive, no extra spaces).
  • Click the "Synchronize" button in the sidebar to update Slides with the latest values from the database.

Do I need to re-link the Sheet every time?

No. Once a database is created or selected, it remains linked to the current Slides file. You can refresh values as often as needed.

What format should I use for variables in Slides?

Use curly brackets: {{ '{{' }}variable_name{{ '}}' }} (e.g., {{ '{{' }}item_name{{ '}}' }}).

What if I accidentally delete the database file?

If the database file is deleted, you can create a new one using the "Create Database" button in the sidebar. Remember to re-add your variables to the new database.

The database file is a normal Sheets file that you can try to recover from the Google Drive trash.

What if I select the wrong database?

If you select the wrong database, simply use the "Select Database" button in the sidebar to choose the correct one. Ensure the correct database is selected before clicking "Synchronize."

Troubleshooting

Add-on not appearing?

Refresh Google Slides™ and ensure you are using the same account you installed the add-on with.

Sidebar won't open or is unresponsive?

Try closing and reopening Google Slides™, or reinstall the add-on if needed.

Unexpected errors or bugs?

Take a screenshot or copy the error message and contact support (details below).

Is the Add-on updated?

Ensure you are using the latest version of the Crystord Add-on. Outdated versions may not support the latest features or workflows. Update it via the Google Workspace Marketplace if needed.
